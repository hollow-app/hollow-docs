# icon.svg Documentation

The `icon.svg` file defines the visual representation of your plugin in Hollow's interface. It is crucial that the icon follows specific guidelines for proper display and styling.

## Key Guidelines for the `icon.svg`:

### 1. No Fill in Paths:

-   Ensure that the `<path>` elements in the SVG do not have any `fill` attribute.
-   The only fill should be applied in the `<svg>` tag itself, allowing Hollow to change the icon's color dynamically.

### 2. Single Color:

-   The icon should be designed using a single color (usually black). This allows Hollow to apply color styling consistently across the app.

### 3. Two Separate Pieces:

-   The icon must contain at least two elements (paths), with one having full opacity (1) and the other at reduced opacity (0.5).
-   This will create a layered effect, with one element standing out and the other slightly transparent.
