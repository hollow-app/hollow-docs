# Styling Plugins: Predefined CSS Classes

Hollow offers several CSS classes for common UI elements, enabling plugin developers to style components effortlessly while maintaining a consistent user experience.

## Button Classes

### `.button-primary`

Styles a primary button that follows the app's primary color scheme.

**Example Usage:**

```html
<button class="button-primary">Primary Button</button>
```

**Showcase:**
<button class="button-primary">Example</button>

### `.button-primary-lite`

A lighter variation of the primary button for less prominent actions.

**Example Usage:**

```html
<button class="button-primary-lite">Lite Primary Button</button>
```

**Showcase:**

<button class="button-primary-lite">Example</button>

### `.button-secondary`

Styles a secondary button that aligns with the app's secondary color scheme.

**Example Usage:**

```html
<button class="button-secondary">Secondary Button</button>
```

**Showcase:**
<button class="button-secondary">Example</button>

### `.button-secondary-lite`

A lighter variation of the secondary button for subtle interactions.

**Example Usage:**

```html
<button class="button-secondary-lite">Lite Secondary Button</button>
```

**Showcase:**

<button class="button-secondary-lite">Example</button>

## Input Classes

### `.input`

Styles a standard input field for forms or text input areas.

**Example Usage:**

```jsx
<input className="input" type="text" placeholder="Enter text here" />
```

**Showcase:**

<input class="input" placeholder="Example" />

### `.selectBox`

Applies styles to dropdown menus, ensuring alignment with Hollow's overall theme.

**Example Usage:**

```jsx
<select className="selectBox">
    <option value="0">Option 1</option>
    <option value="1">Option 2</option>
</select>
```

**Showcase:**

<select className="selectBox">
    <option value="0">Option 1</option>
    <option value="1">Option 2</option>
</select>

### `.slider`

Styles range inputs for a consistent slider design.

**Example Usage:**

```jsx
<input type="range" className="slider" />
```

**Showcase:**

<input type="range" className="slider" />

## Why Use These Classes?

These classes are optimized to ensure a seamless integration with Hollow's design, reducing the need for custom styling. By using them, your plugin's UI will look and feel native to the app.
